## Executive Summary

AgroX is transitioning from a traditional export company website into a **global agricultural trade authority platform**. This document defines the strategic architecture, content strategy, SEO framework, and AI search optimization approach that will make AgroX the definitive digital reference for international buyers of Indian organic agricultural products.

The authority strategy is not about publishing volume. It is about **depth of context** — positioning every AgroX knowledge asset so that buyers, search engines, and AI answer engines all recognize AgroX as the most authoritative, trustworthy, and comprehensive source in the niche.

## 10. AI Search Optimization Strategy

AI answer engines (ChatGPT, Gemini, Perplexity, Claude) pull from:
1. Well-structured HTML with semantic markup
2. Clear question-and-answer text patterns
3. Pages with high domain authority and clear entity signals
4. Content with specific, verifiable facts

### How AgroX Content Must Be Structured for AI Discovery

**Rule 1: Answer before you explain**
Every article's H1 or opening paragraph must directly answer the query that would surface it. Not "In this article we will cover..." but the answer itself.

**Rule 2: Use explicit entity references**
Name real entities: APEDA (not "the government body"), DGFT (not "India's trade authority"), Unjha (not "the main market"), Cuminum cyminum (not "the scientific name").

**Rule 3: Data with attribution**
"India exported 182,000 metric tonnes of cumin in 2023-24 (APEDA)" is AI-citable. "India is a major cumin exporter" is not.

**Rule 4: FAQ schema on every knowledge page**
Every hub page and article must end with 3-5 FAQ schema items covering the most common related questions.

**Rule 5: Structured HowTo markup on process pages**
Export Process, Buyer Guides, Trade Academy articles — all use HowTo schema to feed directly into AI step-by-step answers.

**Rule 6: Consistent entity mentions**
Refer to the company as "AgroX" and "AgroX Organic Exports" consistently. Refer to products by both common name AND scientific name. Refer to certifications by full name AND abbreviation.

## 12. Technical SEO Architecture

### URL Structure Rules
```
/{hub}/{topic-slug}.php
```

All URLs must:
- Be lowercase with hyphens
- Include the primary keyword in the slug
- Avoid dates in slugs (for evergreen content)
- Use dates ONLY for market reports (updates are part of the value)

### Crawlability
- Every hub page must be in sitemap.xml
- Every article must link to its parent hub
- `robots.txt` must NOT block `/knowledge/`, `/buyer-guides/`, `/market-reports/`
- No `noindex` on any authority content page

### Core Web Vitals
- Hero images: AVIF format, explicit width/height, priority loading
- No blocking JS in `<head>`
- CSS animations: `transform` and `opacity` only (GPU-accelerated)
- Font loading: `font-display: swap` on all web fonts
